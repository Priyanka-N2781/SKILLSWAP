Build a full-featured, scalable, modern, 
and user-friendly SkillSwap Android application
 connected to a MySQL database using Java for the frontend and Spring Boot for the backend. Start with a splash screen showing the app logo and name. Implement two separate registration and login flows: registration collects name, email, password, department, year, phone number, and profile picture, with email verification via OTP link and phone number verification via SMS OTP; login allows email/password, Google, or GitHub authentication, includes “Remember Me” and “Forgot Password” options, and uses JWT tokens for secure sessions. Design a MySQL database with tables for users (id, name, email, hashed password, phone, department, year, profile picture, email_verified, phone_verified, points, badges), skills (id, name, description, category, level, image, user_id, created_at), skill_requests (id, skill_id, requester_id, status, created_at, updated_at), sessions (id, skill_id, learner_id, start_date, end_date, status, remarks), feedback (id, session_id, reviewer_id, rating, comment, created_at), and gamification (id, user_id, points, badges). Set up foreign key relationships and indexes for performance. Implement Spring Boot controllers, services, and repositories for user, skill, skill request, session, feedback, and gamification modules with RESTful APIs returning JSON responses. Include input validation, error handling, authentication middleware, role-based access control, and email/SMS verification logic with hashed passwords. The skill module allows adding, viewing, filtering, searching, and sorting skills; skill requests allow sending, accepting/rejecting, and tracking status in real time; session module handles automatic session creation on request acceptance, start/end dates, calendar integration, completion marking, and feedback collection; history module displays all completed sessions, skill requests, points, badges, and allows exporting to PDF. The Android frontend integrates with the backend using Retrofit, handles API responses with loading indicators, error messages, and smooth UI updates. Design UI using Material Design, CardView, RecyclerView, animations, dark/light theme toggle, intuitive navigation, and user-friendly layouts: Splash → Login/Register → Home Dashboard → Add Skill → View Skills → Skill Requests → Sessions → Feedback → History → Profile. Implement notifications for requests, session reminders, and achievements, gamification with points and badges, Lottie animations for milestones, and leaderboards. Ensure modularity with separate packages for activities, models, adapters, network, and utilities. The system should be secure, maintainable, visually attractive, interactive, fully functional, and ready for exam viva, with end-to-end database connectivity and real-time updates for all modules.
